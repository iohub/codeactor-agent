# 任务：性能瓶颈分析

请分析这个 Python 项目的源代码，识别所有性能瓶颈和代码质量问题，生成详细的性能分析报告保存为 `PERF_ANALYSIS.md`。

## 分析范围

扫描所有 `app/` 目录下的 Python 文件，包括：
- `app/api.py` — API 接口层
- `app/processor.py` — 数据处理
- `app/data_loader.py` — 文件加载
- `app/cache.py` — 缓存实现

## 分析维度

1. **时间复杂度问题** — 识别 O(n²) 或更差的算法
2. **空间复杂度问题** — 内存泄漏、不必要的大对象分配
3. **I/O 问题** — 文件读取策略、N+1 查询模式
4. **缓存策略** — 缓存有效性、过期机制、内存泄露
5. **重复计算** — 不必要的重复操作
6. **并发问题** — 线程安全、锁竞争（如有）

## 报告格式

每个问题需包含：
- 文件路径和函数名
- 问题描述和复杂度分析
- 性能影响评估（高/中/低）
- 具体的优化建议（含优化后代码示例）
- 优化优先级排序（P0/P1/P2）

## 验收标准

生成的 `PERF_ANALYSIS.md` 需包含全面的性能分析，识别至少 5 个性能问题。
