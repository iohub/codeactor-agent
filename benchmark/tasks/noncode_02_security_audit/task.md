# 任务：安全漏洞审计

请对这个 Python FastAPI 项目进行全面的安全审计，生成报告保存为 `SECURITY_AUDIT.md`。

## 审计要求

扫描所有源代码文件，识别安全漏洞，按严重程度分类：

### Critical（严重）
- 可直接导致数据泄露或系统被控制的漏洞

### High（高危）
- 可被利用绕过安全控制的漏洞

### Medium（中危）
- 增加攻击面的配置或编码问题

### Low（低危）
- 最佳实践偏离

## 审计维度

1. **认证与授权** — JWT 实现、密码存储、权限验证
2. **数据安全** — SQL 注入、敏感信息泄露
3. **配置安全** — 硬编码密钥、调试模式、CORS 配置
4. **输入验证** — 用户输入校验、类型检查
5. **加密安全** — 密钥管理、算法选择

## 报告格式

每个漏洞需包含：
- 漏洞编号和严重程度
- 文件路径和行号
- 漏洞描述
- 代码片段（关键部分）
- 攻击向量说明
- 具体修复建议（含示例代码）

## 验收标准

生成的 `SECURITY_AUDIT.md` 需包含完整的漏洞分析和修复建议。
