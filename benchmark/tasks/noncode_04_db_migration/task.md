# 任务：制定数据库迁移方案

请分析项目中的旧数据库 schema（`schemas/old_schema.sql`）和新数据库 schema（`schemas/new_schema.sql`），制定完整的数据库迁移方案，保存为 `MIGRATION_PLAN.md`。

## 背景

旧系统使用单一的 `users` 表，角色信息以逗号分隔字符串存储，用户画像字段内嵌在表中。新系统将其规范化为 4 张表：
- `users` — 核心身份信息
- `profiles` — 用户画像（1:1）
- `roles` — 角色定义
- `user_roles` — 用户-角色多对多关系

样例数据在 `seed_data.sql` 中。

## 方案要求

1. **迁移概览** — 新旧 schema 变化对比说明
2. **详细迁移步骤** — 每一步的具体 SQL 语句
   - 创建新表
   - 数据迁移（字段映射、拆分逗号分隔的角色）
   - 数据完整性验证
3. **数据转换映射** — 旧字段到新表/新字段的对应关系表
4. **回滚方案** — 如何撤销迁移恢复到旧 schema
5. **验证方法** — 如何确认迁移成功（行数对比、数据抽样、外键完整性）
6. **风险分析** — 数据丢失风险、停机时间、性能影响

## 验收标准

生成的 `MIGRATION_PLAN.md` 需包含以上所有内容的完整迁移方案。
