# 任务：实现 NFA 正则表达式引擎

请实现一个基于 NFA（非确定有限自动机）的正则表达式匹配引擎。

## 功能要求

1. **编译** `compile(pattern: str)` — 将正则模式编译为 NFA
2. **完全匹配** `match(pattern, text) -> bool` — 模式匹配整个文本
3. **部分匹配** `search(pattern, text) -> bool` — 模式匹配文本的任意子串

## 支持的语法

| 语法 | 含义 |
|------|------|
| 字面量 | 精确匹配字符 |
| `.` | 匹配任意单字符 |
| `*` | 前一个元素零次或多次 |
| `+` | 前一个元素一次或多次 |
| `?` | 前一个元素零次或一次 |
| `|` | 或（交替） |
| `[abc]` | 字符类，匹配其中任意字符 |
| `[^abc]` | 否定字符类 |
| `[a-z]` | 字符范围 |
| `^` | 文本开头锚点 |
| `$` | 文本结尾锚点 |
| `()` | 分组 |
| `\` | 转义特殊字符 |

## 技术约束

- 使用 NFA（Thompson 构造法）或回溯法实现
- 不使用 `re` 标准库
- 正确处理边界情况

## 验收标准

运行 `python3 -m pytest tests/ -v` 所有 15 个测试通过即完成任务。
